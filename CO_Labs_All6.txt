Lab 1 - Module 2: S3 Static Website

- Download pem key, change permissions
- SSH into CLI host: ssh -i labsuser.pem ec2-user@<public-ip>
- Configure AWS CLI: aws configure
- Verify no buckets exist: aws s3 ls
- Create S3 bucket with region: aws s3 mb s3://<bucket-name> --region <region>
- Check IAM user with S3 full access, login as that user in console
- cd into sysops-activity-files, extract gunzip files, delete gunzip folder: cd sysops-activity-files then gunzip <file>.gz
- Edit ownership controls and ACL for bucket: aws s3api put-bucket-ownership-controls --bucket <bucket> --ownership-controls Rules=[{ObjectOwnership=BucketOwnerEnforced}]
- Configure static website hosting, set index.html: aws s3 website s3://<bucket> --index-document index.html
- Upload files to bucket: aws s3 cp . s3://<bucket> --recursive
- Open console, get S3 endpoint URL, test in browser
- Create batch file for repeatable updates
- Update index.html, run batch file, verify changes in browser
- Replace cp with sync in batch file for efficiency: aws s3 sync . s3://<bucket>

Lab 3 - Module 7: Troubleshoot VPC

- SSH into CLI host: ssh -i labsuser.pem ec2-user@<public-ip>
- Configure AWS CLI: aws configure
- Create S3 bucket for flow logs: aws s3 mb s3://<flowlog-bucket>
- Get VPC1 ID: aws ec2 describe-vpcs --filters "Name=tag:Name,Values=VPC1"
- Enable VPC Flow Logs on VPC1: aws ec2 create-flow-logs --resource-type VPC --resource-ids <vpc-id> --traffic-type ALL --log-destination-type s3 --log-destination arn:aws:s3:::<flowlog-bucket>
- Confirm flow log created: aws ec2 describe-flow-logs
- Get web server instance details: aws ec2 describe-instances --query "Reservations[*].Instances[*].[State.Name,PrivateIpAddress,InstanceId,SecurityGroups,SubnetId,KeyName]"
- Try SSH to web server, get timeout
- Check open ports: nmap <web-server-ip>
- Check security group details: aws ec2 describe-security-groups --group-ids <sg-id>
- Check route table, no IGW route found: aws ec2 describe-route-tables
- Get internet gateway ID: aws ec2 describe-internet-gateways
- Create route to IGW, website now loads: aws ec2 create-route --route-table-id <rt-id> --destination-cidr-block 0.0.0.0/0 --gateway-id <igw-id>
- Try SSH again, still fails, check NACL in console, port 22 DENIED
- Delete NACL deny rule for port 22 in console, SSH works now
- Download and analyse flow logs: mkdir flowlogs then cd flowlogs
- List buckets to get name: aws s3 ls
- Download flow logs: aws s3 cp s3://<flowlog####>/ . --recursive
- Extract logs: gunzip *.gz
- See all rejected traffic: grep -rn REJECT .
- Count rejected records: grep -rn REJECT . | wc -l
- Find blocked SSH attempts: grep -rn ' 22 ' . | grep REJECT
- Filter by your IP: grep -rn ' 22 ' . | grep REJECT | grep <ip-address>

Lab 6 - Module 8: Working with S3

- SSH into CLI host: ssh -i labsuser.pem ec2-user@<public-ip>
- Configure AWS CLI: aws configure
- Create S3 bucket in us-east-1: aws s3 mb s3://<bucket-name> --region us-east-1
- Upload sample images to /images path: aws s3 cp <images-folder> s3://<bucket>/images/ --recursive
- List bucket contents: aws s3 ls s3://<bucket>
- IAM console, find mediacouser, go to security credentials tab, create access key, download CSV, copy account number
- Open incognito tab, login as mediacouser using console link, observe limited S3 permissions
- SNS, create Standard topic, edit topic policy, save
- Create subscription, protocol Email, enter email, save, go to inbox and accept subscription
- Create S3 bucket policy JSON file, apply via CLI, receive email confirmation: aws s3api put-bucket-policy --bucket <bucket> --policy file://policy.json
- Update S3 event notification via CLI, receive SNS email: aws s3api put-bucket-notification-configuration --bucket <bucket> --notification-configuration file://notification.json
- Switch to mediacouser credentials: aws configure
- PUT object into bucket, receive SNS email: aws s3api put-object --bucket <bucket> --key <filename> --body <file>
- GET object from bucket: aws s3api get-object --bucket <bucket> --key <filename> <output-file>
- DELETE object from bucket: aws s3api delete-object --bucket <bucket> --key <filename>
- Try changing bucket permissions, access denied as expected: aws s3api put-bucket-acl --bucket <bucket> --acl public-read
